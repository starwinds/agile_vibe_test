# Sprint 2 Retrospective

## 잘된 점 (What went well)
- **계획된 모든 기능 구현:** Sprint 2 백로그에 포함된 모든 기능(하이브리드 검색 API, 데이터베이스 스키마 변경 등)을 성공적으로 구현했습니다.
- **테스트 품질 개선:** Sprint 1 회고의 액션 아이템이었던 `pytest` Fixture를 도입하여 테스트 간 데이터베이스 상태를 완벽하게 격리했습니다. 이는 테스트의 안정성과 신뢰도를 크게 향상시켰습니다.
- **안정적인 개발 환경:** `pytest.ini` 설정, `db_utils` 리팩토링 등을 통해 테스트 환경에서 발생했던 `ModuleNotFoundError` 및 `pgvector` 타입 등록 오류를 근본적으로 해결하여 개발 효율성을 높였습니다.

## 개선할 점 (What could be improved)
- **"완료의 정의(DoD)" 미준수:**
    - **테스트 커버리지:** 목표였던 90%에 약간 못 미치는 89%를 기록했습니다.
    - **문서화:** API 실행 방법 및 사용 예시를 `README.md`에 업데이트하지 못해 문서가 코드 변경 사항을 따라가지 못하는 작은 기술 부채가 발생했습니다.
- **사소한 문서 오류:** `progress.md` 업데이트 과정에서 오탈자가 포함되어 추가 수정이 필요했습니다. 문서 변경 시 조금 더 세심한 검토가 필요합니다.

## 다음 스프린트를 위한 액션 아이템 (Action items for next sprint)
- **`README.md` 문서 최신화:** Sprint 3 시작과 함께 가장 먼저 `README.md`에 Flask 서버 실행 방법과 `curl`을 이용한 API 사용법을 추가하여 문서를 최신 상태로 만듭니다.
- **테스트 커버리지 90% 달성:** API 에러 핸들링 경로 등 테스트가 부족한 부분을 파악하고, 1~2개의 테스트 케이스를 추가하여 목표 커버리지를 달성합니다.
- **문서 검토 프로세스:** 문서 업데이트 시 커밋하기 전 간단한 자체 검토(self-review)를 진행하여 오탈자나 포맷 오류를 방지합니다.
